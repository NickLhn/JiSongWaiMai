# 校园点餐系统开发文档

## 1. 项目概述

### 1.1 项目名称
基于Spring Boot + Vue的校园点餐系统设计与实现

### 1.2 项目背景
随着高校规模的不断扩大，校园餐饮服务面临诸多挑战：就餐高峰期排队拥挤、点餐效率低下、信息不透明等问题日益突出。传统的人工点餐方式已难以满足师生对便捷、高效餐饮服务的需求。本项目旨在设计并实现一套校园点餐系统，实现校园餐饮从点餐、支付、配餐到取餐的全流程数字化管理。

### 1.3 项目目标
- 解决传统点餐排队久、效率低、信息不透明等痛点
- 提升师生校园就餐体验
- 实现校园餐饮运营的数字化、智能化管理
- 支持学生端、商家端、管理员端三端协同

## 5. 接口设计规范

### 5.1 接口命名规范

- 基础路径：`/api/v1`
- RESTful风格：
  - GET：查询操作
  - POST：新增操作
  - PUT：更新操作
  - DELETE：删除操作

### 5.2 核心接口列表

#### 5.2.1 用户模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 用户登录 | POST | /auth/login | 用户登录认证 |
| 用户注册 | POST | /auth/register | 用户注册 |
| 获取用户信息 | GET | /user/info | 获取当前用户信息 |
| 更新用户信息 | PUT | /user/info | 更新用户信息 |

#### 5.2.2 菜品模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 菜品列表 | GET | /dishes | 获取菜品列表(支持分页、筛选) |
| 菜品详情 | GET | /dishes/{id} | 获取菜品详情 |
| 菜品搜索 | GET | /dishes/search | 搜索菜品 |
| 添加菜品 | POST | /merchant/dishes | 商家添加菜品 |
| 更新菜品 | PUT | /merchant/dishes/{id} | 商家更新菜品 |
| 删除菜品 | DELETE | /merchant/dishes/{id} | 商家删除菜品 |

#### 5.2.3 订单模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 创建订单 | POST | /orders | 创建订单 |
| 订单列表 | GET | /orders | 获取用户订单列表 |
| 订单详情 | GET | /orders/{id} | 获取订单详情 |
| 取消订单 | PUT | /orders/{id}/cancel | 取消订单 |
| 支付订单 | POST | /orders/{id}/pay | 支付订单 |
| 商家接单 | PUT | /merchant/orders/{id}/accept | 商家接单 |
| 更新配送状态 | PUT | /merchant/orders/{id}/deliver | 更新配送状态 |

#### 5.2.4 购物车模块

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 购物车列表 | GET | /cart | 获取购物车 |
| 添加购物车 | POST | /cart | 添加商品到购物车 |
| 更新数量 | PUT | /cart/{id} | 更新购物车商品数量 |
| 删除商品 | DELETE | /cart/{id} | 删除购物车商品 |
| 清空购物车 | DELETE | /cart | 清空购物车 |

### 5.3 统一响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1707782400000
}
```

## 10. 安全设计

### 10.1 认证授权
- 采用JWT Token进行身份认证
- 基于角色的权限控制(RBAC)
- 接口权限校验

### 10.2 数据安全
- 用户密码加密存储(BCrypt)
- 敏感信息传输加密(HTTPS)
- SQL注入防护
- XSS攻击防护

### 10.3 接口安全
- 接口签名验证
- 请求频率限制
- 参数校验
